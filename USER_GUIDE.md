# 高中数学错题诊断系统快速入门指南

这份文档面向第一次使用当前系统的老师，目标是让你能在本机快速启动系统，并完成一条完整教学闭环：创建学生、录入错题、人工校对、查看诊断、生成练习包、记录复习结果。

## 1. 系统能做什么

当前系统是一个本地运行的高中数学错题诊断工具，适合单个老师管理多个学生。数据默认保存在本机，不需要把学生错题或图片上传到外部服务。

核心功能包括：

- 学生档案管理
- 错题文字和图片录入
- 题干、答案、解析、订正提示四栏校对
- 苏教版高中数学知识点标注
- 错误类型归因
- 学生诊断看板
- 错题复习窗口和掌握度记录
- 按薄弱知识点生成专项练习包
- 教材题库选题、收藏、停用、备注和打印
- 教案中心沉淀解释方式和题目解法
- 本地教材 PDF 识别和题源候选确认

## 2. 第一次启动

打开 PowerShell，进入项目目录：

```powershell
Set-Location D:\桌面\教学系统
```

如果是第一次使用，或刚更新过代码，按顺序执行：

```powershell
npm.cmd install
npm.cmd run setup
npm.cmd run dev
```

看到命令行出现 `Ready` 或 `http://localhost:3000` 后，在浏览器打开：

```text
http://localhost:3000/login
```

开发服务窗口需要保持打开。关闭窗口后，网页服务也会停止。

## 3. 登录账号

默认示例账号：

- 教师姓名：`示例教师`
- 登录标识：`demo`

登录后会进入“学生工作台”。如果你改用自己的姓名和登录标识，系统会在本地创建或复用对应教师账号。

## 4. 日常启动和停止

日常使用时通常只需要：

```powershell
Set-Location D:\桌面\教学系统
npm.cmd run dev
```

然后打开：

```text
http://localhost:3000/login
```

停止系统时，在运行 `npm.cmd run dev` 的 PowerShell 窗口按 `Ctrl + C`。如果出现确认提示，输入 `Y` 后回车。

## 5. 推荐使用流程

### 第一步：创建学生

进入“工作台”后，在右侧“新增学生”区域填写：

- 姓名
- 年级，默认是高三
- 学校，可不填

点击“保存学生”。保存后，学生会出现在工作台的“学生”区域。

### 第二步：录入错题

点击左侧导航或工作台顶部的“录入错题”。

在错题草稿页选择学生，然后填写或上传以下内容：

- 题干草稿
- 答案草稿
- 解析草稿
- 订正提示

每个草稿区都支持文字和多张图片。你也可以先只上传图片，后面再到校对页补文字。

可选信息：

- 题型，例如选择题、填空题、解答题
- 年份
- 错误类型
- 知识点

点击“保存并进入校对”。

### 第三步：人工校对入库

错题保存后会进入“错题校对”页。这里是系统统计质量最关键的一步。

建议至少补齐：

- 题干
- 错因或订正提示
- 错误类型
- 知识点
- 复习日期

确认无误后点击：

- “保存校对结果”：保存并停留在当前页
- “保存并校对下一题”：连续处理待校对错题
- “保存后回学生档案”：保存并回到当前学生

校对完成后，错题状态会从“待校对”变成“已入库”，诊断、复习和练习包生成才会使用它。

### 第四步：查看学生诊断

进入某个学生档案后，可以点击“诊断看板”。

诊断页会按错题沉淀：

- 重复薄弱知识点
- 错误类型分布
- 待复习错题
- 近期趋势

如果诊断为空，通常说明该学生还没有完成校对入库的错题。

### 第五步：生成练习包

在学生档案或诊断页点击“生成练习包”或“按薄弱项出练习”。

系统会优先根据学生薄弱知识点，从本地教材题源里选题。进入练习包编辑页后，你可以：

- 修改练习包标题
- 编辑题目、答案和解析
- 添加自定义题目
- 打开“题库选题”
- 按教材、模块、章节、知识点筛选
- 收藏、停用或备注教材题源
- 调整题目顺序
- 删除题目
- 保存或确认练习包
- 打印学生卷、答案解析或完整版本

建议生成后先检查题目，再点击“确认”。

### 第六步：记录复习结果

已入库错题的校对页会出现“完成一次复习”区域。

学生复习后，选择：

- “忘记了”
- “部分掌握”
- “已掌握”

也可以填写复习备注。保存后，系统会更新知识点掌握度，并计算下次复习窗口。

## 6. 复习窗口怎么理解

工作台上可以切换老师的复习模式：

- 上学期：默认只在周末释放复习任务
- 寒暑假：按更短间隔释放复习任务

学生档案里可以设置：

- 可登录频率：每周末、隔周周末、每月周末、假期集中
- 每次题量：控制一次释放多少题

掌握度大致变化规则：

- 忘记了：掌握度下降，下次更快复习
- 部分掌握：小幅提升，短期复习
- 已掌握：明显提升，连续掌握后间隔变长

## 7. 教材识别和题源

项目根目录目前包含四本苏教版高中数学 PDF：

- `苏教版高中数学 必修第1册.pdf`
- `苏教版高中数学 必修第2册.pdf`
- `苏教版高中数学 选择性必修1.pdf`
- `苏教版高中数学 选择性必修2.pdf`

系统初始化时会写入教材目录和知识点。如果本机有可用的 PDF 文本抽取工具，`npm.cmd run setup` 或 `npm.cmd run db:seed` 会尝试抽取教材练习题源。

在左侧导航进入“教材识别”，可以：

- 查看各教材已识别页数和内容块数量
- 点击“重跑本地教材识别”
- 筛选教材内容块
- 检查低置信题源候选
- 将候选题源确认为正式题库题目
- 忽略不合适的候选

如果题源不足，练习包会生成“请打开教材某章节选题”的占位题源。此时可以先手工编辑练习包，也可以重跑教材识别或重新执行 `npm.cmd run db:seed`。

## 8. 教案中心

左侧导航进入“教案中心”。

这里按知识点沉淀教学材料，适合记录：

- 某个知识点的解释方式
- 某道题的不同解法
- 学生贡献的好方法
- 涉及的背景知识

进入知识点后，可以新增“解释方式”或“题目解法”，也可以绑定教材题源。内容积累后可直接打印教案。

## 9. 数据保存位置

常见数据位置：

- 数据库：`prisma/dev.db`
- 错题图片：`uploads/`
- 教材 OCR 临时文件：`tmp/textbook-ocr/`
- 环境变量：`.env`
- 示例配置：`.env.example`

如果要备份系统数据，优先备份：

- `prisma/dev.db`
- `uploads/`
- `.env`

安装依赖：

```powershell
npm.cmd install
```

初始化数据库、同步表结构并写入示例数据：

```powershell
npm.cmd run setup
```

启动开发服务：


## 11. 常见问题

### 浏览器打不开 `localhost:3000`

先确认 `npm.cmd run dev` 的 PowerShell 窗口还开着，并且已经显示 `Ready`。如果窗口已经关闭，重新运行：

```powershell
npm.cmd run dev
```

### 提示端口 3000 被占用

通常是之前的开发服务还没关。先回到旧 PowerShell 窗口按 `Ctrl + C` 停止；找不到旧窗口时，可以重启电脑后再启动。

### `setup` 还没结束，可以直接打开网页吗

不建议。`setup` 负责准备数据库和示例数据，等命令结束后再运行 `npm.cmd run dev`。

### 出现 Prisma DLL、EPERM 或 `query_engine-windows.dll.node` 相关错误

先关闭正在运行的 `npm.cmd run dev` 或 `npm.cmd run start` 窗口，再重新执行失败的命令。

### 上传错题后诊断没有变化

检查错题是否还处于“待校对”。只有保存校对结果、进入“已入库”的错题，才会参与诊断、复习和练习包生成。

### AI 任务显示“已跳过”

当前 `.env` 默认是：

```text
AI_PROVIDER="none"
AI_API_KEY=""
```

这表示系统不会调用外部 AI 服务，相关任务会安全跳过。

## 12. 最短上手路径

如果只想快速体验一遍，按这个顺序走：

1. `npm.cmd run dev`
2. 打开 `http://localhost:3000/login`
3. 用 `示例教师` / `demo` 登录
4. 在工作台新建一个学生
5. 点击“录入错题”，填写一段题干或上传图片
6. 保存后在校对页补知识点和错误类型
7. 保存校对结果
8. 回到学生档案查看诊断
9. 点击“生成练习包”
10. 在练习包页编辑、确认并打印
